# Agent Skills 规范

技能（Skill）是一个包含指令、脚本和资源的文件夹，代理可以动态发现和加载这些内容，以便在特定任务中表现更好。要使文件夹被识别为技能，它必须包含一个 `SKILL.md` 文件。

# 技能文件夹布局

一个最小的技能文件夹如下所示：

```
my-skill/
  - SKILL.md
```

更复杂的技能可以根据需要添加额外的目录和文件。


# SKILL.md 文件

技能的"入口点"是 `SKILL.md` 文件。这是唯一必须存在的文件。该文件必须以 YAML 前置元数据（frontmatter）开头，后跟常规 Markdown 内容。

## YAML 前置元数据

YAML 前置元数据有 2 个必需属性：

- `name`
    - 技能的名称，使用连字符命名法（hyphen-case）
    - 仅限于小写 Unicode 字母数字和连字符
    - 必须与包含 SKILL.md 的目录名称匹配
- `description`
    - 描述技能的功能以及 Claude 应该何时使用它

有 3 个可选属性：

- `license`
    - 应用于技能的许可证
    - 我们建议保持简短（可以是许可证名称或捆绑的许可证文件名称）
- `allowed-tools`
    - 预先批准运行的工具列表
    - 目前仅在 Claude Code 中支持
- `metadata`
    - 从字符串键到字符串值的映射
    - 客户端可以使用它来存储 Agent Skills 规范未定义的额外属性
    - 我们建议使您的键名具有合理的唯一性，以避免意外冲突

## Markdown 正文

Markdown 正文没有任何限制。
